# Site Foundation - Project Brief

## Identita del progetto

Site Foundation e la base madre privata per creare siti web professionali top di gamma.

Non e un sito finale.
Non e un template pubblico.
Non e un progetto da pubblicare online.

E una fondazione tecnica privata, generica, sicura, ordinata e riutilizzabile.

## Obiettivo

Creare una base solida da cui partire per futuri siti web, come:

- hotel
- ristoranti
- liberi professionisti
- aziende locali
- portfolio
- landing page
- attivita commerciali
- progetti web piu complessi

## Principio fondamentale

La base deve contenere poche cose, ma fatte molto bene.

Ogni elemento deve essere:

- generico
- riutilizzabile
- stabile
- facile da modificare
- difficile da rompere
- adatto a progetti professionali

## Cosa appartiene alla base

- struttura del progetto
- configurazione centrale
- componenti UI fondamentali
- layout base
- SEO base
- accessibilita base
- responsive base
- documentazione
- regole di qualita
- metodo Git pulito

## Cosa NON appartiene alla base

- contenuti specifici per hotel
- menu ristorante
- prenotazioni
- cataloghi immobili
- testi commerciali definitivi
- sezioni troppo specifiche
- funzioni legate a un solo settore

## Regola di lavoro

Ogni modifica deve seguire questo ciclo:

1. modifica locale
2. controllo visivo su localhost
3. pnpm lint
4. pnpm build
5. git status
6. git add
7. git commit
8. git push su GitHub privato

## Pubblicazione

Site Foundation non deve essere collegato a Vercel e non deve avere un deploy pubblico.

Vercel verra usato solo per i progetti reali derivati da questa base.
