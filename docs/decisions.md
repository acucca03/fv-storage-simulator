# Decisions

## Decisione 1 - Progetto privato

Site Foundation deve restare privato.

Motivo: e una base strategica personale per costruire futuri siti professionali.

## Decisione 2 - Nessun deploy pubblico

Il progetto non deve essere pubblicato online.

Motivo: non e un sito finale, ma una fondazione privata.

## Decisione 3 - Stack tecnico

Usiamo:

- Next.js
- TypeScript
- Tailwind CSS
- pnpm
- Git
- GitHub privato
- Cursor
- WSL Ubuntu

## Decisione 4 - Base neutra

La base deve rimanere neutra e non legata a un settore specifico.

Motivo: deve poter diventare un sito per hotel, ristoranti, professionisti, aziende o altri progetti.

## Decisione 5 - Poche cose, fatte bene

Non aggiungiamo funzioni solo per riempire il progetto.

Ogni elemento deve aumentare solidita, ordine o riutilizzabilita.

## Decisione 6 - Configurazione centralizzata

Nomi, link, navigazione, tema e metadata devono essere gestiti da file centrali.

Motivo: i futuri siti devono essere personalizzabili senza modificare decine di componenti.

## Decisione 7 - Qualita prima della velocita

Ogni modifica deve passare da lint, build e controllo Git.

Motivo: la base deve essere affidabile.
