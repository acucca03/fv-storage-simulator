# Pre Project Checklist

## Obiettivo

Prima di creare un nuovo sito derivato da Site Foundation, bisogna chiarire il progetto.

Questa checklist evita di partire con codice prima di avere obiettivi chiari.

## Identita progetto

- [ ] Nome del sito
- [ ] Settore
- [ ] Obiettivo principale
- [ ] Pubblico target
- [ ] Tono comunicativo
- [ ] Lingua o lingue
- [ ] Dominio previsto

## Contenuti

- [ ] Logo disponibile
- [ ] Palette colori
- [ ] Font desiderati
- [ ] Immagini disponibili
- [ ] Testi principali
- [ ] Servizi/prodotti da mostrare
- [ ] Contatti
- [ ] Link social
- [ ] Privacy/cookie se necessari

## Struttura sito

- [ ] Home
- [ ] Chi siamo
- [ ] Servizi
- [ ] Gallery
- [ ] FAQ
- [ ] Contatti
- [ ] Altre pagine specifiche

## Funzionalita

- [ ] Form contatto
- [ ] Prenotazioni
- [ ] Ordini
- [ ] Pagamenti
- [ ] Login
- [ ] Dashboard
- [ ] Area admin
- [ ] Newsletter
- [ ] Multilingua

## Scalabilita

- [ ] Traffico previsto basso, medio o alto
- [ ] Presenza di dati dinamici
- [ ] Necessita di database
- [ ] Necessita di API
- [ ] Necessita di cache
- [ ] Necessita di monitoraggio errori

## Decisione finale

Prima di iniziare il codice, deve essere chiaro:

- cosa si sta costruendo
- per chi
- con quali funzionalita
- cosa va pubblicato
- cosa deve restare privato
